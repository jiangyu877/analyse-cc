# Magic Bento 功能矩阵设计

## 背景

经营概览首页当前使用固定左侧功能栏。用户选择将首页功能入口改为 Magic Bento 矩阵，以获得更清晰的模块层级和更接近参考界面的视觉效果。项目是 Flask/Jinja 应用，不引入 React 或前端构建链。

## 目标

- 仅在经营概览首页移除左侧功能栏，并在 Hero 与经营数据之间增加 Bento 功能矩阵。
- 保留现有顶部导航、动态 LineWaves 背景、所有业务路由和管理员权限判断。
- 使用深色卡片、克制的紫色边缘辉光和现有薄荷绿色品牌色。
- 在桌面、平板和手机上保持文字清晰、入口稳定、无重叠和无横向滚动。
- 即使 JavaScript 不可用，所有功能入口仍可正常导航。

## 非目标

- 不重构客户、商品、订单等业务页面的导航。
- 不引入 React、GSAP、OGL 或新的 CDN 依赖。
- 不修改后端数据模型、路由、权限或业务算法。
- 不增加大幅倾斜、磁吸位移或持续循环动画。

## 页面结构

经营概览首页按以下顺序排列：

1. 现有顶部导航。
2. 现有 Hero、背景图片和 LineWaves。
3. 新增“业务功能”Bento 矩阵。
4. 现有经营趋势、最近订单和其他经营数据。

首页不再渲染 `.home-sidebar`、移动端侧栏遮罩和对应的开关逻辑。其他业务页面保持原样。

## 功能卡片

普通用户看到以下入口：

| 卡片 | 路由 | 视觉权重 |
| --- | --- | --- |
| 客户 360 | `customers.index` | 标准卡片 |
| 商品库存 | `products.index` | 标准卡片 |
| 订单业务 | `orders.index` | 横向卡片 |
| 支付记录 | `payments.index` | 标准卡片 |
| 退款管理 | `refunds.index` | 标准卡片 |
| 算法任务 | `algorithms.index` | 重点大卡片 |

管理员额外看到：

| 卡片 | 路由 | 视觉权重 |
| --- | --- | --- |
| 数据导入 | `imports.index` | 标准卡片 |
| 只读 SQL | `custom_query.custom_query_page` | 标准卡片 |

每张卡片是完整可点击的 `<a>` 元素，包含 Tabler 图标、业务标签、标题、一句说明和右上角箭头。管理员卡片继续使用现有 `session.get('role') == 'admin'` 条件渲染。

## 视觉系统

- 卡片背景接近用户参考代码中的 `#120f17`，但使用页面现有设计令牌统一维护。
- 默认边框为低对比中性色；圆角固定为 8px。
- 紫色只用于鼠标附近的边缘辉光和少量粒子，不作为大面积背景。
- 薄荷绿色继续用于关键状态、图标或小型强调，避免页面成为单一紫色主题。
- 卡片标题与描述保持紧凑，说明最多两行并提供可预测的最小高度。
- 不在卡片内嵌套卡片，也不使用装饰性渐变球体。

桌面端采用四列网格。算法任务卡片跨两列和两行，订单业务跨两列。其余卡片由 CSS Grid 自动排布，因此管理员卡片存在或缺失时都不会留下固定空洞。

## 交互

新增独立原生脚本 `app/static/js/magic-bento.js`：

- 在指针移动时，通过 `requestAnimationFrame` 更新卡片的 `--glow-x`、`--glow-y` 和 `--glow-intensity`。
- 在矩阵范围内显示一个不拦截点击的全局聚光层。
- 仅在桌面精确指针设备上，为当前悬停卡片生成少量、短生命周期粒子。
- 键盘 `:focus-visible` 使用与悬停一致的清晰边框反馈。
- 脚本初始化失败时不影响链接、布局或权限控制。

在 `prefers-reduced-motion: reduce`、触摸设备和窄屏设备上关闭粒子与追踪动画，仅保留静态焦点和悬停样式。所有事件监听器只绑定在 Bento 根节点，并在页面卸载时释放临时节点和动画帧。

## 响应式规则

- 大于等于 1024px：四列 Bento 布局，使用非对称跨列卡片。
- 600px 至 1023px：两列布局，重点卡片跨两列，卡片不使用固定纵向跨行。
- 小于 600px：单列布局，所有卡片等宽，标题和说明自然换行。
- 所有断点都使用稳定网格轨道和最小高度，悬停、图标和动态内容不得引起布局位移。

## 可访问性与容错

- 功能矩阵使用带可访问名称的 `<nav>`；每张卡片使用语义链接。
- 装饰性粒子和聚光层标记为隐藏并设置 `pointer-events: none`。
- 颜色不是唯一状态提示；焦点同时使用边框和阴影。
- 外部 Tabler 图标字体加载失败时，文字标题和说明仍完整可用。
- JavaScript 关闭或 WebGL 不可用时，Bento 导航与现有页面内容仍然可用。

## 测试与验收

自动化测试先定义并验证以下行为：

- 首页渲染 Bento 根节点、六个普通入口及正确路由。
- 管理员入口继续受现有模板权限条件保护。
- 首页不再渲染旧侧栏结构和侧栏开关脚本。
- 样式包含四列、两列和单列断点，以及减少动态效果规则。
- 新 JavaScript 文件通过 Node 语法检查，模板通过 Flask 实际渲染。
- 完整现有测试套件没有回归。

视觉验收使用桌面和手机尺寸截图，确认：

- 页面无横向滚动、文字截断或组件重叠。
- 非管理员和管理员卡片数量都能自然排布。
- 背景动画、Bento 辉光与正文对比度保持清晰。
- 鼠标、键盘和触摸设备均可可靠进入所有功能。
